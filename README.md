# ObsidianFlow

> **Production-grade Next.js 14 boilerplate** with authentication, role-based access, admin panel, and a self-hosted setup wizard.

---

## Features

| Area | Details |
|---|---|
| **Auth** | RS256 JWT (access + refresh), httpOnly cookies, bcrypt passwords, account lockout, email verification |
| **Database** | MongoDB via Mongoose вЂ” singleton connection, AES-256-GCM field-level encryption, soft-delete plugin |
| **Roles** | Fully configurable roles from wizard вЂ” per-role permissions stored in DB |
| **Admin Panel** | Standalone local UI (port 3002) for User management, role CRUD, system config |
| **Setup Wizard** | 7-step React wizard at `localhost:3001` вЂ” generates `project.config.ts` from your choices |
| **Security** | Rate limiting, endpoint guard (disable any route at runtime), CSP headers, no source maps in prod |
| **Anti-Debug** | Optional вЂ” DevTools detection, console poisoning, React DevTools hook poisoning |
| **Performance** | Local cache warming sidecar (port 2999) вЂ” starts automatically with dev server |
| **Design System** | CSS variables (primary/accent/dark mode), theme injected at runtime from config |

## Quick Start

### 1 вЂ” Prerequisites

- Node.js в‰Ґ 20
- A MongoDB connection string (local or Atlas)

### 2 вЂ” Install root dependencies

```bash
npm install
```

### 3 вЂ” First run (launches setup wizard)

```bash
node start.js
```

The wizard at `http://localhost:3001` guides you through:
1. Project metadata & branding
2. Auth settings (identifier, password rules, session limits)
3. Registration fields (enable firstName, username, phone, etc.)
4. Roles & default role
5. Theme (primary color, accent, dark mode, font)
6. Security settings (rate limits, lockout, anti-debug)
7. Review & initialise (seeds DB, writes `project.config.ts`, writes `.setup-done`)

After the wizard, the app automatically reloads at `http://localhost:3000`.

### 4 вЂ” Subsequent runs

```bash
node start.js        # detects .setup-done, starts next.js directly
# or
cd main-app && npm run dev
```

## Environment Variables

Copy `.env.example` в†’ `main-app/.env.local` and fill in values:

```bash
cp .env.example main-app/.env.local
```

| Variable | Required | Description |
|---|---|---|
| `MONGODB_URI` | вњ… | MongoDB connection string |
| `JWT_PRIVATE_KEY` | вњ… | RS256 PEM private key (generate below) |
| `JWT_PUBLIC_KEY` | вњ… | RS256 PEM public key |
| `AES_ENCRYPTION_KEY` | вњ… | 32-byte key as 64-char hex or base64-44 |
| `NODE_ENV` | в¬њ | `development` (default) or `production` |

**Generate RS256 key pair:**
```bash
openssl genrsa -out private.pem 2048
openssl rsa -in private.pem -pubout -out public.pem
# Then paste the contents (with \n escaped) into .env.local
```

**Generate AES key:**
```bash
openssl rand -hex 32
```

## Auth Flow

```
Register в†’ POST /api/auth/register
  в†’ Zod validation (schema from projectConfig)
  в†’ Duplicate check (email/username)
  в†’ bcrypt hash (cost 12)
  в†’ Create User + Session
  в†’ Set __refresh httpOnly cookie
  в†’ Return { accessToken, user }

Login в†’ POST /api/auth/login
  в†’ loginWithEmail provider
  в†’ Lockout check (5 attempts в†’ 15 min)
  в†’ bcrypt verify
  в†’ Max concurrent session enforcement
  в†’ Create Session (tokenHash stored, not token)
  в†’ Set __refresh httpOnly cookie
  в†’ Return { accessToken, user }

Token refresh в†’ POST /api/auth/refresh
  в†’ Read __refresh cookie
  в†’ Verify RS256 signature
  в†’ rotateRefreshToken (invalidate old session, create new)
  в†’ Return new accessToken, rotate cookie
```
